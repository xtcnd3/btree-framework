    btree-framework

    The btree-framework is aimed at programmers that are looking for associative data container classes capable of handling data sets at a very large scale. Said data containers are coming with an interface similar to the existing STL container classes map, multimap, set and multiset. Furthermore, iterators, with a random access qualifier enabled, are supported, which ought to allow for a quick interchange between this framework and existing STL containers. Also, an array type, that isn't suffering from any slow down when data is inserted or removed at the beginning of the list, is available. This array type has an interface similar to the STL class vector. All container types can be abstracted and allow applications to create their own container type hierarchy.

    btree-framework

    • any container provided, may it be an associative and a linear type, is based on a btree structure
      • which is supposed to deliver increased performance, since other implementations use linear arrays, link lists and red-black as well as binary trees
      • any of the above structures developes poor performance when it comes to large data amounts, while btree can scale quite well to that end
    • any container type can be abstracted, as oppose to the existing STL equivalents
      • allowing applications to create their own container type hierarchy
      • also the sorting criteria is a virtual method and not a template parameter
    • supports scary iterator assignments, as described by Bjarne Sproustrup’s paper “Minimizing Dependencies within Generic Classes for Faster and Smaller Programs”
    • iterators have random tag capability
    • can handle very large amounts of data (64 bit address space)
      • with other STL implementations, the programmer depends what size_t is set to, while with this framework the address space type can be set
    • cmake support

    How To Build

    This project supports cmake, which ought to make it very easy to include the resulting btree-framework library in any project that uses cmake.

    How to build the windows btree-framework test bench for x86 and x86_64 platforms?

    Via cygwin command execute the following commands:

    cd .../to/your/projects/location
git clone https://github.com/andreas-steffens/btree-framework.git
cd btree-framework/src/btreetest

    Here you can find a number of batch files called msvce201x.x86(_64).bat. These batch files can be executed via command line or windows explorer. Each of those batch files creates a directory named "cmake" and creates all cmake files and visual studio express solution files, set up for the selected platform and visual studio express version, in there. Then it tries to open whatever the visual studio express application can be found under the registery key: HKEY_CLASSES_ROOT\Applications\WDExpress.exe\shell\open\command

    msvce2013.x86.bat    - Visual Studio Express 2013 with platform Win32
msvce2013.x86_64.bat - Visual Studio Express 2013 with platform x64
msvce2015.x86.bat    - Visual Studio Express 2015 with platform Win32
msvce2015.x86_64.bat - Visual Studio Express 2015 with platform x64

    If the project needs re-opening at a later point, simply use the same batch file again. In the event that you select the wrong batch file, cmake will refuse to create new project files and aborts with an error message. As a result your existing project files remain unharmed. In case you want build the project say for a different platform, then you need to close the visual studio instance that has the btree-framework project open. After that, you need to delete or rename the afore mentioned cmake directory and execute a different batch file that generates the project files for the desired platform. However, any modifications that have been made to the project will be lost!

      ## Version

    The current version of the project is: 0.05 (alpha)

    Please don't be discouraged by the fact that the version contains the word "alpha". What this means is that the feature set is incomplete (see below).

    • emplace ()
    • emplace_hint ()
    • initializer_list<> support
    • move constructors and move assignment operators
    • equal_range ()
